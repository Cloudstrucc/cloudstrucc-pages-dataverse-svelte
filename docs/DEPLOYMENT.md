# Deployment and ALM

## Supported package lifecycle

The unpacked source tree in this repository is unmanaged-only. The supported lifecycle is:

1. Pack unmanaged locally.
2. Import unmanaged into a development Dataverse environment.
3. Publish customizations and run tests.
4. Run Solution Checker.
5. Export authoritative unmanaged and managed packages from Dataverse.
6. Promote the exported managed package to test and production.

Do not attempt to create a managed package directly from this unmanaged-only source with `pac solution pack --packagetype Managed`. PAC will reject the mismatch.

## Build unmanaged packages

macOS/Linux:

```bash
./scripts/build-solutions.sh
```

PowerShell:

```powershell
./scripts/Build-Solutions.ps1
```

Outputs:

```text
solution/schema/packed/CloudstruccPagesSchema_1_0_2_0_unmanaged.zip
solution/full/packed/CloudstruccPagesStudio_1_0_2_0_unmanaged.zip
```

## Import into development

```bash
./scripts/import-solution.sh \
  --environment-url https://YOURORG.crm3.dynamics.com \
  --solution-path ./solution/full/packed/CloudstruccPagesStudio_1_0_2_0_unmanaged.zip \
  --settings-file ./config/deployment-settings.dev.json
```

## Export validated artifacts

```bash
./scripts/export-validated-solution.sh \
  --environment-url https://YOURORG.crm3.dynamics.com \
  --solution-name CloudstruccPagesStudio \
  --output-dir ./solution/full/packed
```

The export script writes both unmanaged and managed packages generated by Dataverse.

## Deploy unmanaged development package

```bash
./scripts/deploy.sh \
  --environment-url https://YOURORG.crm3.dynamics.com \
  --package-type unmanaged \
  --settings-file ./config/deployment-settings.dev.json
```

## Deploy managed package

```bash
./scripts/deploy.sh \
  --environment-url https://YOURORG.crm3.dynamics.com \
  --package-type managed \
  --solution-path ./solution/full/packed/CloudstruccPagesStudio_managed.zip \
  --settings-file ./config/deployment-settings.prod.json
```

## Refresh source from Dataverse

After validating the solution in Dataverse, export the unmanaged package and unpack it over the source-controlled solution folder:

```bash
pac solution unpack \
  --zipfile ./solution/full/packed/CloudstruccPagesStudio_unmanaged.zip \
  --folder ./solution/full/unpacked \
  --packagetype Unmanaged \
  --clobber
```

This step replaces bootstrap XML with the authoritative sharded metadata generated by Dataverse.

## Warning about `Customizations.xml`

If PAC reports that `Entities` has unexpected children, it may skip table-specific processing. Do not promote that package to an important environment until you have completed a successful development import/export cycle and refreshed the unpacked source from the Dataverse-generated export.

## Recommended pipeline

1. Lint and test source.
2. Build Svelte applications.
3. Copy built files into solution web resources.
4. Pack unmanaged.
5. Import into an integration environment.
6. Run smoke tests and Solution Checker.
7. Export validated unmanaged and managed solutions.
8. Publish the managed artifact.
9. Import with deployment settings.

## Rollback

- Use solution upgrades for managed releases.
- Keep the previous managed version available.
- Store page/component revisions in Dataverse before publishing.
- Use a holding solution for staged upgrades where appropriate.
